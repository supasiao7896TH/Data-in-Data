# รีวิวโค้ด — Boardman KB App (`project/Boardman KB App.dc.html`)

รีวิวนี้ครอบคลุมแอป "คลังความรู้ Boardman" (Phase ที่เป็นแอปจริงตามที่คุยใน `chats/chat1.md`)
ตรวจทั้งความถูกต้องเชิงตรรกะ (logic), พฤติกรรม runtime ของ DC framework (`project/support.js`)
และความสอดคล้องกับ requirement หลักของผู้ใช้

> สรุปสั้น: ฟังก์ชันหลัก (เพิ่ม/ค้นหา/ปักหมุด/แก้/ลบ/เก็บ localStorage) ทำงานได้จริง
> แต่มี **bug ระดับ blocker 1 ข้อ** (เรื่องออฟไลน์) และ bug ตรรกะ/UX อีกหลายข้อ
> ข้อที่ฟิกซ์แล้วในรอบนี้ทำเครื่องหมาย ✅ FIXED ข้อที่ยังต้องตัดสินใจทำเครื่องหมาย ⚠️ ต้องคุยต่อ

## 🔴 Critical

### C1. แอปพึ่ง CDN ทั้งหมด — ขัดกับ requirement "ใช้ในเครื่อง ไม่ต้องต่อเน็ต" ⚠️ ต้องคุยต่อ
ข้อความแรกของผู้ใช้: _"...ทำงานในเครื่องคุณ ไม่ต้องต่อเน็ต"_ และห้องควบคุมโรงงานปิโตรเคมีส่วนใหญ่
**ไม่มีอินเทอร์เน็ต** แต่ตอนนี้ทุกชิ้นส่วนสำคัญโหลดจากเน็ต:

| สิ่งที่โหลดจากเน็ต | แหล่ง | ถ้าออฟไลน์ |
|---|---|---|
| React / ReactDOM | `unpkg.com` (`support.js:1377-1380`) | **แอปไม่บูตเลย — จอว่าง** |
| ฟอนต์ IBM Plex | `fonts.googleapis.com` (บรรทัด 11-13) | ตกไปใช้ฟอนต์ระบบ (พอรับได้) |
| อ่าน Excel (xlsx) | `cdn.sheetjs.com` (บรรทัด 14) | ลากไฟล์ Excel ไม่ได้ |
| อ่าน PDF (pdf.js) | `cdnjs.cloudflare.com` (บรรทัด 15, 357) | ลากไฟล์ PDF ไม่ได้ |
| OCR (tesseract) | `cdn.jsdelivr.net` (บรรทัด 503) | ปุ่ม "อ่านข้อความจากภาพ" พัง |

ผลคือถ้าเอาไฟล์ไปเปิดในเครื่องที่ห้องควบคุมจริง (ไม่มีเน็ต) **แอปจะเปิดไม่ขึ้นด้วยซ้ำ**
ส่วนที่ออฟไลน์จริงคือแค่ "การเก็บข้อมูล" (localStorage) เท่านั้น

**แนวทางแก้ (ต้องเลือกก่อนทำ):**
- (ก) ดาวน์โหลด React/ReactDOM/xlsx/pdf.js/tesseract + ฟอนต์ มาวางในโฟลเดอร์เดียวกัน แล้วชี้ path เป็นไฟล์ในเครื่อง (self-contained 1 โฟลเดอร์ก๊อปใส่ USB ได้)
- (ข) ถ้าจะ implement ใหม่จริง ให้ bundle ทุกอย่างเป็นไฟล์เดียว (เช่น Vite + เก็บ asset ในตัว)
- (ค) ถ้ามี intranet ในโรงงาน ให้ host ไลบรารีบน server ภายใน

> รบกวนยืนยันว่าเครื่องที่ใช้งานจริงมีเน็ตหรือไม่ ก่อนผมลงมือทำส่วนนี้ (เป็นงานใหญ่กว่าฟิกซ์ bug)

## 🟠 Major (logic bugs)

### M1. เลือกหมวดเองแล้วถูกระบบเขียนทับ ✅ FIXED
`blankForm()` ตั้ง `category: 'setpoint'` เป็นค่าเริ่มต้น แล้วโค้ดเดิมใช้เงื่อนไข
`f.category !== 'setpoint'` เป็นตัวเช็คว่า "ผู้ใช้เลือกหมวดหรือยัง"
ปัญหา: `'setpoint'` เป็นทั้ง **ค่า default** และ **หมวดจริงที่เลือกได้** — ชนกัน

ผลคือ ถ้าผู้ใช้กดเลือกหมวด **"Setpoint / Alarm"** เองแล้วค่อยลากไฟล์/กด OCR เข้ามา
ระบบจะ "เดาหมวดใหม่" ทับของที่เลือกไว้เงียบ ๆ (เช่นเด้งไปเป็น interlock) — ผิดความตั้งใจผู้ใช้

**ฟิกซ์:** เพิ่ม flag `categoryTouched` ใน form, ตั้ง `true` เมื่อผู้ใช้กดเลือกชิปหมวด
แล้วเปลี่ยน `applyParsed()` / `runOCR()` ให้เดาหมวดเฉพาะตอน `!categoryTouched`
(ตอนแก้ไขรายการเดิม ถือว่า touched = true เพราะหมวดเดิมตั้งใจไว้แล้ว)

### M2. กล่อง overlay ตอนลากไฟล์ค้างคาจอ ✅ FIXED
เดิม `_dragleave` ซ่อน overlay เฉพาะเมื่อ `e.clientX <= 0 && e.clientY <= 0`
(คือต้องลากออกทาง **มุมซ้ายบนเป๊ะ ๆ** เท่านั้น) — บนจอใหญ่ห้องควบคุม
ถ้าลากไฟล์เข้ามาแล้วเปลี่ยนใจลากออกทางขวา/ล่าง/ปล่อยนอกหน้าต่าง overlay
"↧ วางไฟล์เพื่อเพิ่มเข้าคลัง" จะค้างทับทั้งจอ

**ฟิกซ์:** เช็ค `e.relatedTarget == null` หรือเคอร์เซอร์ออกนอกขอบ viewport ทุกด้าน
และเพิ่ม listener `dragend` เป็นตาข่ายกันค้างอีกชั้น

## 🟡 Minor / UX (ยังไม่ฟิกซ์ — รอยืนยันแนวทาง)

### m3. หัวข้อ "ทั้งหมด" นับไม่รวมรายการปักหมุด ⚠️
ในหน้า "ทั้งหมด" ที่มีของปักหมุด: รายการปักหมุดถูกแยกขึ้นไปโชว์ section บน
ส่วน header ด้านล่างยังเขียนว่า "ทั้งหมด" แต่ตัวเลข `resultCount` นับเฉพาะรายการ
ที่ **ไม่** ปักหมุด (เพราะ `listSource` ตัดตัวที่ pinned ออก — บรรทัด ~613)
เช่นมี 6 รายการ ปักหมุด 2 → มุมขวาบนขึ้น "6 รายการ" แต่ header ล่างขึ้น "ทั้งหมด · 4 รายการ"
สับสน แนะนำเปลี่ยน label เป็น "รายการอื่น" หรือให้ตัวเลขนับรวม

### m4. การ paste รูปดักไว้ทั้งหน้า อาจแย่ง paste ของช่องค้นหา/textarea ⚠️
`onPaste` ผูกไว้ที่ `document` ทั้งหน้า ถ้า clipboard มี "ภาพ" จะเด้งเปิด modal เพิ่มข้อมูล
ทันที + `e.preventDefault()` ผลคือถ้าผู้ใช้กำลังพิมพ์ในช่องค้นหา/เนื้อหาแล้ว clipboard
ดันมีภาพติดมาด้วย จะ paste ข้อความลงช่องไม่ได้ และเด้ง modal แทน
แนะนำ: ข้ามพฤติกรรมนี้เมื่อโฟกัสอยู่ในช่อง input/textarea (ยกเว้น dropzone)

### m5. controlled input ใช้ `onInput` ไม่มี `onChange`
React จะเตือน warning ใน console ("provided a `value` prop without `onChange`")
ใช้งานได้ปกติ แต่ควรเพิ่ม `onChange` คู่กัน หรือใช้ `defaultValue` เพื่อตัด warning

### m6. ลากหลายภาพพร้อมกัน เก็บแค่ภาพสุดท้าย
`handleFiles()` วน loop ภาพ แต่ละภาพ set `form.image` ทับกัน → แนบได้ทีละ 1 ภาพ
ถ้าตั้งใจให้ 1 รายการมีหลายภาพต้องเปลี่ยนเป็น array (ตอนนี้ดีไซน์รองรับ 1 ภาพ — ถือว่าโอเค)

### m7. โค้ดเล็ก ๆ ที่ควรเก็บกวาด
- `loadItems()` อ่าน `localStorage.getItem('bkb_items_v2')` เป็น string ตรง ๆ
  ขณะที่ `persist()` ใช้ค่าคงที่ `this.STORE_KEY` — ค่าตรงกัน แต่ควรใช้ `STORE_KEY` ให้ทั้งคู่ กันพลาดตอนแก้ key
- ไฟล์ที่ไม่ใช่ภาพ/Excel/PDF จะถูกอ่านด้วย `readText` ดื้อ ๆ (เช่น .docx จะได้ binary เพี้ยน) — ควรเตือนชนิดไฟล์ที่ไม่รองรับ

## ✅ จุดที่ตรวจแล้วว่า "ไม่ใช่บั๊ก" (false alarm)
- คำเตือน "file not found" ของ `{{ form.image }}` / `{{ detail.image }}` เป็น false positive จริง
  (เป็น `<img src>` ที่อยู่ใน `sc-if` และ resolve เป็น data-URL ตอน runtime เท่านั้น)
- `b.pinned - a.pinned` ใช้ boolean ลบกัน — JS แปลงเป็น 1/0 ทำงานถูกต้อง
- `togglePin` หยุด event propagation ถูกแล้ว กดดาว ★ บนการ์ดไม่เผลอเปิดหน้า detail
- ลากหลายไฟล์ที่เป็นเอกสาร: content ต่อกันถูกต้อง (เพราะ `setState` ของ DC อัปเดต `logic.state` แบบ sync)
